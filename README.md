# 🛡️ TrustLens AI

## The AI That Verifies AI

TrustLens AI is an Explainable AI Trust Platform built using Microsoft Azure AI Foundry.

Traditional AI systems generate answers but rarely explain whether those answers should be trusted. TrustLens solves this problem by transforming every AI response into a transparent trust report backed by evidence, source analysis, consensus evaluation, freshness scoring, and hallucination risk assessment.

Instead of asking:

> "What is the answer?"

TrustLens asks:

> "Can this answer be trusted?"

---

# 🚀 Problem Statement

Organizations increasingly rely on AI for:

* Business decision making
* Compliance guidance
* Software development
* Research and analysis
* Healthcare assistance
* Document intelligence

However, AI systems often operate as black boxes.

Users receive answers without knowing:

* Why the answer was generated
* Whether sources support it
* Whether the information is current
* Whether sources agree with each other
* Whether contradictory information exists
* Whether the answer may be hallucinated

This creates trust, compliance, and governance challenges for enterprises.

---

# 💡 Our Solution

TrustLens AI acts as an independent trust layer on top of AI-generated responses.

For every answer generated by the AI agent, TrustLens automatically evaluates:

* Trust Score
* Evidence Strength
* Source Freshness
* Consensus Level
* Contradiction Detection
* Hallucination Risk
* Human Review Recommendation

This allows users to make informed decisions rather than blindly trusting AI outputs.

# ☁️ Microsoft Azure AI Foundry Integration

TrustLens is powered by Microsoft Azure AI Foundry.

### Azure AI Foundry Agent

Our TrustLens Verifier Agent uses GPT-4.1-mini for reasoning and answer generation.

### Grounded Web Search

The agent performs real-time evidence retrieval using Azure Foundry's Bing-powered grounding capabilities.

### Knowledge Base

Azure AI Foundry Knowledge is used to provide contextual grounding and source retrieval.

### GPT-4.1-mini

Provides cost-efficient reasoning and enterprise-grade response generation.

---

# 🔍 Trust Engine

The Trust Engine is the core innovation of TrustLens.

After an answer is generated, TrustLens independently evaluates the response.

### Evidence Analysis

Measures supporting evidence strength.

### Consensus Analysis

Determines whether sources agree.

### Contradiction Detection

Identifies conflicting evidence.

### Freshness Analysis

Measures how recent supporting information is.

### Hallucination Detection

Estimates the likelihood of unsupported claims.

# 📊 Trust Metrics

Every response includes:

### Trust Score

Overall confidence score (0-100).

### Evidence

Strength of supporting evidence.

### Consensus

Agreement between sources.

### Freshness

Recency of information.

### Hallucination Risk

Likelihood of unsupported claims.

### Supporting Sources

Sources that reinforce the answer.

### Contradicting Sources

Sources that challenge the answer.

### Neutral Sources

Sources that are related but inconclusive.

# 🛠️ Installation

## Frontend

```bash
cd frontend

npm install

npm run dev
```

Frontend runs on:

```text
http://localhost:3000
```

---

## Backend

```bash
cd backend

pip install -r requirements.txt

uvicorn main:app --reload
```

Backend runs on:

```text
http://localhost:8000
```

Swagger:

```text
http://localhost:8000/docs
```

---

# 🔑 Environment Variables

Create a `.env` file in the backend directory:

```env
PROJECT_ENDPOINT=YOUR_ENDPOINT

MODEL_NAME=YOUR_MODEL

AZURE_OPENAI_API_KEY=YOUR_KEY
```

Do not commit `.env` files to GitHub.
